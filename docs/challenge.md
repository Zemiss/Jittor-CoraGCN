# 计图挑战赛赛道一热身赛说明文档

## 一、赛题介绍

本赛道将使用 **JittorGeometric 图神经网络库**，在基于 **Cora** 的引文网络数据集上训练 **GCN（Graph Convolutional Network）模型**，完成节点分类任务。

参赛选手需要基于给定的数据集与示例代码框架，训练图神经网络模型，并对测试集中的节点类别进行预测。

---

## 二、赛题内容

本赛题提供一个基于 Cora 的引文网络数据集：

```text
data/cora.pkl
```

该数据集已包含在发布包中。参赛选手需要基于 **JittorGeometric** 库，使用 **GCN 模型** 完成节点分类任务。

最终需要对测试集节点预测其所属类别，并将预测结果保存为指定格式的 JSON 文件。

---

## 三、任务说明

### 1. 输入

输入为引文网络图数据：

```text
data/cora.pkl
```

该数据文件已经包含在比赛发布包中。

### 2. 输出

输出为测试集节点的预测分类结果，保存为 JSON 文件：

```text
result.json
```

提交格式为一个字典：

- `key`：测试集节点编号
- `value`：预测类别编号

需要注意的是，虽然题目说明中节点编号为 `int`，但提交示例中 JSON 的 key 使用字符串形式。因此建议在生成 `result.json` 时，将节点编号统一保存为字符串。

示例格式如下：

```json
{
    "1708": 0,
    "1709": 1,
    "1710": 3
}
```

### 3. 注意事项

数据集中测试集节点的标签被隐藏，统一标记为 `-1`。参赛选手需要通过训练模型对这些节点进行预测。

---

## 四、数据集简介

本赛题使用的数据集基于经典的 **Cora 引文网络数据集**。

### 1. 基本信息

| 项目 | 数值 | 说明 |
|---|---:|---|
| 节点数 | 2708 | 每个节点代表一篇论文 |
| 边数 | 5429 | 边代表论文之间的引用关系 |
| 特征维度 | 1433 | 词袋特征 |
| 类别数 | 7 | 节点分类类别数量 |

### 2. 数据字段说明

数据集以 pickle 格式提供，包含以下字段：

| 字段名 | 类型 / 形状 | 说明 |
|---|---|---|
| `x` | `numpy array, shape: 2708 × 1433` | 节点特征矩阵 |
| `y` | `numpy array, shape: 2708` | 节点标签。训练集和验证集标签完整，测试集标签为 `-1` |
| `edge_index` | `numpy array, shape: 2 × num_edges` | 图的边列表 |
| `train_mask` | mask | 训练集节点掩码 |
| `val_mask` | mask | 验证集节点掩码 |
| `test_mask` | mask | 测试集节点掩码 |
| `num_classes` | int | 类别数 |
| `num_features` | int | 特征维度 |

## 五、数据集与示例代码

本赛题提供数据集与示例代码框架，包含以下基础功能：

- 训练步骤

参赛选手可以基于示例代码，补全注释中标记为 `TODO` 的部分，从而完成该赛题。

## 六、评测方式

评测程序会加载选手提交的预测结果文件：

```text
result.json
```

随后将预测结果与测试集真实标签进行对比，并计算分类准确率：

```text
Accuracy
```

通过标准为：

```text
测试集准确率 ≥ 0.70
```

只要测试集准确率达到或超过 0.70，即视为通过热身赛。

## 八、注意事项

参赛过程中需要注意以下要求：

1. 比赛需要使用 **计图（Jittor）深度学习框架** 以及 **JittorGeometric 图神经网络库**。
2. 更多 JittorGeometric 资料可以参考其官方仓库。
3. 所有团队，包括热身赛和正式赛题，都需要按照开源指南进行代码开源，才能视为有效成绩。具体要求详见赛事介绍。

## 九、任务目标总结

本热身赛的核心目标是：

> 使用 JittorGeometric 在 Cora 引文网络数据集上训练 GCN 模型，完成测试集节点分类，并提交符合格式要求的 `result.json` 文件。

参赛选手需要完成以下主要步骤：

1. 读取 `data/cora.pkl` 数据；
2. 构建 GCN 节点分类模型；
3. 使用训练集训练模型；
4. 在验证集上调参并评估模型；
5. 对测试集节点进行预测；
6. 生成 `result.json`；
7. 将 `gcn.py` 与 `result.json` 打包为 `result.zip` 提交。
